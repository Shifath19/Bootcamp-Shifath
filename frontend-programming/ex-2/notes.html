Exercise 2: Using Tools

Objective:

Learn how to package and structure projects using TypeScript while following a structured development life cycle: Development → Testing → Production.

Steps Followed:

  • Set Up a TypeScript Project: Initialized the project using npm and configured TypeScript with a tsconfig.json file.
  • Solved Problems:
    • Problem 1: Loaded a JSON file and dynamically rendered a table with sorting functionality.
    • Problem 2: Parsed CSV data using PapaParse, enhanced table filtering, and row highlighting.
    • Problem 3: Integrated a build tool (tsc) and packaged the app for development and production.
  • Development Life Cycle: Used vite for:
    • Development server with live reload.
    • Linting and type-checking during development.
    • Production builds with bundling.

Challenges Faced:

  • Understanding and configuring tsconfig.json for complex setups.
  • Debugging issues while fetching and parsing CSV data using PapaParse, especially with malformed data rows.
  • Ensuring compatibility of bundled code across different environments (development vs. production).

Outputs:

  • Successfully rendered a table with JSON data and implemented sorting functionality.
  • Integrated CSV parsing and filtering using PapaParse with visual row highlights for specific conditions.
  • Packaged the application for production with minimized and optimized assets.

Experience:

This exercise was an enriching experience that provided practical exposure to TypeScript project structuring, development workflows, and tool integration. Overcoming challenges, especially around bundling and environment-specific configurations, significantly deepened my understanding of modern web development tools.
