Abstract

Use the debug package for selective debugging in multi-module scenarios.

Challenges faced

Setting up the Debug Package: Initially, I struggled with setting up the debug package in my project. I wasn't sure how to configure it for selective debugging, especially in multi-module scenarios. I spent some time figuring out how to set the DEBUG environment variable correctly so that debug messages would appear in the terminal.
Managing Debug Namespaces for Multiple Modules: As I extended the program to have multiple modules, each with its own debug namespace, it became tricky to control which module's messages were displayed. I had to learn how to use the DEBUG environment variable to selectively enable debugging for specific modules without cluttering the terminal output.
