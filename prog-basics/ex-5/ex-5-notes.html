Abstract

Managing Environment Variables (dotenv)

Challenges faced

Handling sensitive information like API keys and database credentials was one of the more challenging aspects. I had to ensure that these details were not accidentally logged or exposed in error messages or debugging logs. This meant careful consideration of how to manage these sensitive variables in a way that maintains security and doesn't expose them to unintended parties.

Improvements

One of the improvements could be Environment-Specific Configuration: I could enhance my configuration by creating separate .env files for different environments (e.g., .env.dev, .env.prod) and using a package like dotenv-expand to manage multiple .env files more efficiently. This would make it easier to handle environment-specific variables.
