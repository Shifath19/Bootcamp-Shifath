YAML Configuration (config)

Challenges faced

The most difficult part was using the configuration to adjust the behavior of the program dynamically. For instance, switching logging levels or loading commands/functions dynamically based on the YAML configuration required thinking about how to decouple the code logic from the configuration. I had to explore dynamic imports and learn how to handle them in a Node.js environment.

Improvements

One of the improvements I could refactor the program to provide a more centralized configuration management system. Instead of using multiple configuration files and manually switching environments, I could create a function that loads and merges the necessary environment configurations automatically. This would make the code cleaner and more modular.
